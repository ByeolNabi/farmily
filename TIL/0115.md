# Jira 주요 용어 가이드

## 1. 작업 단위 (Issue Type)

    * 에픽 (Epic)

    여러 개의 스토리와 태스크를 포함하는 가장 큰 작업 단위
    한 번의 스프린트로 끝나지 않는 큰 규모의 기능 의미
    예시: 서비스 회원 시스템 구축, 식물 성장 기록 기능


    * 스토리 (Story)

    사용자 관점에서 필요한 기능을 설명한 단위
    사용자 가치 중심의 서술 권장
    예시: 식물 사진 업로드 및 가족 공유 기능

    * 태스크 (Task)

    사용자 기능과 직접적 관련은 없으나 프로젝트 진행에 필요한 기술적 업무
    예시: 데이터베이스 설계, 서버 환경 설정, API 문서 작업

    * 하위 작업 (Sub-task)

    스토리나 태스크 완료를 위해 쪼개놓은 가장 작은 실행 단위
    예시: 사진 업로드 UI 개발, 이미지 리사이징 로직 구현, 테스트 코드 작성


## 2. 관리 용어

    * 백로그 (Backlog)

    프로젝트에서 수행해야 할 모든 작업의 리스트
    진행 계획 수립 전의 대기 상태 이슈 모음


    * 스프린트 (Sprint)

    정해진 기간(보통 1~2주) 동안 집중하여 작업을 완료하는 반복 주기
    스토리 포인트 (Story Point)
    작업 시간 대신 업무의 복잡도와 어려움을 나타내는 상대적 난이도 점수


## 3. 보드 구성

    * 스크럼(Scrum): 계획된 주기에 맞춘 작업 완료 방식

    * 칸반(Kanban): 지속적인 업무 흐름 시각화 및 병목 현상 파악 방식
