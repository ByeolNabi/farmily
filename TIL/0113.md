# 협업에 필요한 git 명령어 공부하기

1. 브랜치 관리 및 이동

    팀 프로젝트에서는 각자 맡은 기능별로 가지(Branch)를 쳐서 작업.

        git checkout -b <branch-name>: 브랜치 생성과 동시에 이동하기

        git branch -a: 로컬과 서버에 있는 모든 브랜치 목록 확인하기.

        git branch -d <branch-name>: 작업이 끝난 로컬 브랜치 삭제하기.

        git remote update: 서버의 최신 브랜치 상태를 내 컴퓨터에 업데이트하기 (브랜치가 안 보일 때 유용).


2. 코드 동기화 및 합치기

    남의 코드를 가져오거나 내 코드를 합칠 때 사용.

        git fetch: 서버의 변경 사항을 확인만 하고 가져오지는 않기 (안전한 탐색).

        git pull --rebase: 내 커밋 기록을 서버의 최신 기록 뒤로 붙여서 히스토리를 깔끔하게 관리하기.

        git merge --no-ff <branch-name>: 브랜치를 합친 기록을 명확히 남기며 합치기.

        git cherry-pick <commit-hash>: 다른 브랜치에 있는 특정 커밋 하나만 쏙 뽑아서 내 브랜치에 적용하기.


3. 실수 복구 및 작업 되돌리기

    협업 중 실수는 치명적일 수 있으니 이 명령어들은 반드시 숙지하기.

        git reset --hard HEAD~1: 최근 1개의 커밋을 아예 없었던 일로 만들기 (주의해서 사용).

        git reset --soft HEAD~1: 커밋은 취소하되, 내가 작성한 코드는 남겨두기.

        git revert <commit-hash>: 이미 서버에 올라간 커밋을 취소하는 '취소 커밋' 만들기 (협업 시 권장).

        git checkout -- <file-name>: 특정 파일의 수정을 취소하고 마지막 커밋 상태로 되돌리기.


4. 임시 저장 및 상태 확인

    작업 중에 갑자기 다른 브랜치로 가야 할 때.

        git stash: 하던 작업을 잠시 주머니에 넣어두고 워킹 디렉토리를 깨끗하게 만들기.

        git stash pop: 주머니에 넣어둔 작업 다시 꺼내오기.

        git diff: 내가 수정한 코드와 이전 코드의 차이점을 상세히 보기.

        git status: 현재 어떤 파일이 수정되었고, 어떤 단계에 있는지 확인하기.


5. 히스토리 파악 및 고급 분석

    누가, 언제, 왜 이 코드를 짰는지 알아내는 명령어입니다.

        git log --oneline --graph --all: 전체 브랜치의 흐름을 한눈에 보기 좋게 출력하기.

        git blame <file-name>: 파일의 각 라인을 누가 언제 수정했는지 이름표 붙여서 보기 (범인 찾기용).

        git show <commit-hash>: 특정 커밋에서 어떤 내용이 바뀌었는지 자세히 보기.

        git clean -fd: 추적되지 않는(untracked) 파일이나 폴더 한꺼번에 삭제하기.

---

💡협업할 때는 git pull 보다는 git fetch로 먼저 상황을 살피고,내 코드를 올리기 전에 **git pull --rebase**를 생활화하면 히스토리가 꼬이는 것을 많이 방지할 수 있다.
